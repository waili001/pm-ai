# Project Glossary

This document defines the key terms, entities, and acronyms used within the **PM-AI** project.

## Core Entities

### TP (Touch Point Project)
-   **Definition**: Represents a specific project or initiative tracked in the system.
-   **Source**: Synced from Lark Base (Table: `tp_projects`).
-   **Key Identifiers**: `ticket_number` (e.g., TP-123), `record_id`.
-   **Key Attributes**:
    -   `department`: The owning department.
    -   `participated_dept`: Other departments involved in the project.
    -   `jira_status`: Current lifecycle status (e.g., Open, In Progress, Closed).
    -   `completed_percentage`: Overall progress of the project.

### TCG (TCG Ticket)
-   **Definition**: A development task or work item, typically synced from Jira.
-   **Source**: Synced from Lark Base (Table: `tcg_tickets`).
-   **Key Identifiers**: `tcg_tickets` (e.g., TCG-456), `record_id`.
-   **Relationship**: Can be a child of a TP (`tp_number`). Can have sub-tasks.
-   **Attributes**: `status`, `assignee`, `reporter`, `issue_type` (Bug, Story, Task).

### Program
-   **Definition**: A higher-level grouping of multiple TPs.
-   **Source**: Synced from Lark Base (Table: `tp_program`).
-   **Attributes**: `program_title`, `tp` (List of associated TPs).

### Ticket Anomaly
-   **Definition**: A detected irregularity or issue with a ticket's state that requires attention (e.g., Parent is "In Progress" but all children are "Open").
-   **Source**: Generated by the system based on validation rules (Table: `ticket_anomalies`).
-   **Attributes**: `anomaly_reason`, `detected_at`.

### Member
-   **Definition**: An employee or system user details synced from HR/Lark systems.
-   **Source**: Lark Base (Table: `member_info`).
-   **Attributes**: `name`, `department`, `position`, `team`.

## Access Control (RBAC)

### Admin User
-   **Definition**: A user account for logging into the PM-AI system.
-   **Types**: `LOCAL` (Database auth), `LARK` (Lark OAuth).
-   **Attributes**: `username`, `role_id`.

### Role
-   **Definition**: A named set of permissions assigned to a user (e.g., `SUPER_ADMIN`, `USER`, `PM`, `Manager`).
-   **Function**: Determines what pages and APIs a user can access.

### Page Permission
-   **Definition**: Authorization to access a specific frontend route or page component.
-   **Identifier**: `code` (e.g., `PAGE_PROJECT_PLANNING`).

## Systems & Integrations

### Lark (Feishu)
-   **Role**: Primary data source for Projects, Tickets, and Member info. The system syncs data *from* Lark Base tables.

### Jira
-   **Role**: The issue tracking system. Ticket status and details usually originate here before being synced to Lark and then to PM-AI.
